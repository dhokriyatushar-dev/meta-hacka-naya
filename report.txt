  1. PROJECT OVERVIEW
═══════════════════════════════════════════════

EduPath AI is an OpenEnv-compliant reinforcement learning (RL) environment designed for personalized tutoring. It acts as a bridge between high-quality educational resources and individual student goals, using an AI agent (Tutor) to navigate a multi-field curriculum.

The platform is designed for the Meta Hackathon, ensuring 100% compliance with the OpenEnv specification and the OpenAI Client requirement.

═══════════════════════════════════════════════
  2. DETAILED FEATURE IMPLEMENTATIONS
═══════════════════════════════════════════════

[1] ROADMAP GENERATION
    - TECH STACK: Python, FastAPI, OpenAI Client, Next.js (Frontend)
    - METHODOLOGY: Uses a zero-shot LLM approach (`ai/roadmap_generator.py`) to synthesize a personalized learning curriculum.
    - LOGIC: Takes the student's target field, learning goal, self-assessed skills, parsed Job Description (JD) skills, and weekly hours as input context. The LLM generates a highly structured JSON array of weeks, learning objectives, and mini-projects. It skips topics the user already knows well (based on the onboarding step) and bridges gaps towards the JD.
    - FALLBACK: If the LLM is unavailable, defaults to a rule-based generator that uses the static topic graph from `curriculum.py` to auto-populate weeks sequentially.

[2] COURSE RECOMMENDATION & FETCHING
    - TECH STACK: DuckDuckGo Search API (`duckduckgo-search`), FastAPI, Python `asyncio`
    - METHODOLOGY: Real-time search and caching layer (`ai/resource_fetcher.py`).
    - LOGIC: For any given topic on the roadmap, the backend fires parallel asynchronous search queries targeting high-quality educational domains (Kaggle.com, FreeCodeCamp, fast.ai, HuggingFace, MIT OCW). It extracts the title, URL, and a text snippet, detects the source type, and estimates the duration.
    - TRACKING: Links clicked are tracked in the `StudentProfile`. A student must click at least one resource link before they are allowed to mark the topic as "Studied", unlocking the quiz.
    - CACHING: Results are stored in a local JSON cache valid for 7 days to reduce API calls and latency.

[3] ADAPTIVE QUIZZES
    - TECH STACK: OpenAI Client, Next.js (Interactive UI)
    - METHODOLOGY: Dynamic Multiple-Choice Question (MCQ) generation (`ai/quiz_generator.py`).
    - LOGIC: When a student unlocks a quiz, the LLM generates a 3-5 question quiz tailored to the current topic and the student's skill level.
    - EVALUATION & ADAPTATION:
        * 70-100% (Pass): The topic is marked complete. The student's internal skill level is boosted by +0.3.
        * 50-69% (Partial): The student is given a "Retry" prompt. They retain their current topic but are prompted to review materials.
        * <50% (Fail): The topic completion is rejected, the quiz streak is broken, and alternative learning resources (e.g. interactive notebooks instead of raw documentation) are recommended by the engine.

[4] CAPSTONE PROJECTS & MINI-PROJECTS
    - TECH STACK: Next.js (Dashboard tracking), Python (Environment simulation)
    - METHODOLOGY: Practical application milestones embedded directly in the curriculum.
    - LOGIC: The RL environment defines `assign_mini_project` and `assign_capstone` actions. Mini-projects are recommended every 2-3 topics to consolidate local skills (e.g., "Build a CLI tool"). Capstones are comprehensive (e.g., "Full-Stack App") requiring multiple prerequisite skills. Completion significantly boosts the Job Readiness Score.

[5] JOB READINESS EVALUATION
    - TECH STACK: Python (Algorithmic scoring in `student.py`)
    - METHODOLOGY: A continuous 0-100% weighted score indicating labor-market readiness.
    - LOGIC: The formula dynamically aggregates multiple vectors:
        * 40% Topic Completion Volume (normalized against the target role's required skills).
        * 30% Average Quiz Score (accuracy metric).
        * 30% Project Completion (practical application).
    - If the user uploaded a Job Description during onboarding, the algorithm penalizes missing JD-specific skills, ensuring the score accurately reflects readiness for that exact role.

[6] INTERVIEW PREP MATCHING
    - TECH STACK: FastAPI, React State
    - METHODOLOGY: Gated progression system (`api/career.py`).
    - LOGIC: The Interview Preparation module remains locked until the student reaches a 70% Job Readiness Score. Once unlocked, it triggers an LLM-driven mock interview simulation where the AI agent acts as a hiring manager for the user's specific Target Field, asking technical questions mapped to their completed topics.

[7] JOBS, COMPETITIONS, CONFERENCES & EVENTS
    - TECH STACK: Python (Recommendation Agent)
    - METHODOLOGY: Action-space triggering within the OpenEnv engine.
    - LOGIC: As the student progresses, the RL agent can execute the `suggest_event_or_hackathon` action. Based on the student's field (e.g., "Tech" -> Hackathons; "Healthcare" -> Medical Conferences; "Data" -> Kaggle Competitions), the backend serves relevant URLs and event types to the frontend dashboard's Career Tab to encourage real-world networking.

[8] BADGES & MILESTONES
    - TECH STACK: Python deterministic rule-engine (`api/badges.py`)
    - METHODOLOGY: Gamification layer tightly coupled to state-changes.
    - LOGIC: The system listens to state updates on the `StudentProfile`.
        * "Scholar": Awarded on first topic completion.
        * "Hat Trick": Awarded upon reaching a quiz_streak of 3.
        * "Architect": Awarded upon finishing the first Capstone.
    - Badges are permanently persisted in the `backend/data/` JSON storage.
